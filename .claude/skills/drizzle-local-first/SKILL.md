# Drizzle + expo-sqlite — expert

## Why this stack

- **Drizzle**: type-safe SQL builder. No runtime overhead, no decorators, no ActiveRecord magic.
- **expo-sqlite**: Expo's SQLite binding with `enableChangeListener` for live queries.
- Together: fully typed schema, reactive queries, no extra runtime, free.

Alternatives considered:
- WatermelonDB: mature but heavier API, observable-based, harder to reason about types.
- Legend-State + Supabase plugin: trendy, automatic sync, but newer and less control.
- PowerSync: official Supabase recommendation but adds infrastructure cost.

For a free-local + premium-cloud template: **Drizzle wins on simplicity and zero-cost.**

## Schema patterns

### Synced table (mirrors Supabase)

```ts
import { sql } from "drizzle-orm";
import { integer, sqliteTable, text } from "drizzle-orm/sqlite-core";

export const items = sqliteTable("items", {
  id: text("id").primaryKey(),                              // local UUID
  serverId: text("server_id"),                              // null until sync confirms
  title: text("title").notNull(),
  isDone: integer("is_done", { mode: "boolean" }).notNull().default(false),
  createdAt: integer("created_at", { mode: "timestamp_ms" }).notNull()
    .default(sql`(unixepoch() * 1000)`),
  updatedAt: integer("updated_at", { mode: "timestamp_ms" }).notNull()
    .default(sql`(unixepoch() * 1000)`),
  dirty: integer("dirty", { mode: "boolean" }).notNull().default(true),
  deletedAt: integer("deleted_at", { mode: "timestamp_ms" }),
});

export type Item = typeof items.$inferSelect;
export type NewItem = typeof items.$inferInsert;
```

**Why `mode: "timestamp_ms"`**: stores ms since epoch as integer. Lighter than ISO strings, sorts correctly, syncs predictably with Postgres `timestamptz`.

**Why `mode: "boolean"`**: SQLite has no native boolean — Drizzle stores `0`/`1`. Without `mode: "boolean"`, you'd be comparing to `0`/`1` everywhere.

### Local-only table (never syncs)

Skip the sync metadata fields:

```ts
export const preferences = sqliteTable("preferences", {
  key: text("key").primaryKey(),
  value: text("value").notNull(),
});
```

## ID generation

Don't use SQLite `autoincrement` for synced tables — IDs collide across devices. Generate locally:

```ts
function newId(prefix = "local"): string {
  return `${prefix}_${Date.now()}_${Math.random().toString(36).slice(2, 8)}`;
}
```

For deterministic IDs (e.g. when importing), use `crypto.randomUUID()` from `expo-crypto`.

## Migrations

```bash
npm run db:generate   # drizzle-kit generates SQL diff in drizzle/
npm run db:migrate    # NOT used at runtime — runs once on app start via ensureMigrations()
```

The template's `lib/db/client.ts` calls `migrate(db, migrations)` from `drizzle-orm/expo-sqlite/migrator`. The `migrations` import comes from `drizzle/migrations.ts`, generated by drizzle-kit.

After `db:generate`, you'll see:
```
drizzle/
  0000_curly_jubilee.sql
  meta/
    _journal.json
    0000_snapshot.json
  migrations.ts          ← imports the .sql files
```

Commit ALL files in `drizzle/` — these are your migration history.

## Querying

### Reactive (recommended for screens)

```tsx
import { useLiveQuery } from "drizzle-orm/expo-sqlite";
import { eq, isNull } from "drizzle-orm";

const { data, error } = useLiveQuery(
  db.select().from(items).where(isNull(items.deletedAt)).orderBy(items.createdAt)
);
```

`useLiveQuery` hooks into `enableChangeListener` and re-runs on any insert/update/delete to the queried tables. Use this in components.

### Imperative (for actions)

```ts
await db.insert(items).values({ id: newId(), title: "..." });

await db.update(items)
  .set({ isDone: true, updatedAt: Date.now(), dirty: true })
  .where(eq(items.id, id));

await db.delete(items).where(eq(items.id, id));   // hard delete
// or soft:
await db.update(items)
  .set({ deletedAt: Date.now(), dirty: true })
  .where(eq(items.id, id));
```

### Joins

```ts
db.select({
  itemId: items.id,
  itemTitle: items.title,
  tagName: tags.name,
})
.from(items)
.leftJoin(itemTags, eq(itemTags.itemId, items.id))
.leftJoin(tags, eq(tags.id, itemTags.tagId));
```

## Transactions

```ts
await db.transaction(async (tx) => {
  await tx.insert(items).values({ id, title });
  await tx.insert(itemTags).values({ itemId: id, tagId });
});
```

If any throws, the whole tx rolls back. Use for any multi-write operation.

## Sync metadata invariants

Every write to a synced table:
- Insert: `dirty = true` (default), `serverId = null`, `updatedAt = now`.
- Update: `dirty = true`, `updatedAt = Date.now()`.
- Soft delete: `deletedAt = Date.now(), dirty = true`.
- After server upsert: `dirty = false, serverId = <returned>`.

The sync engine in `lib/db/sync.ts` does the last step. Don't manually flip `dirty` outside of it.

## Common errors

- **"no such table: items"** → migrations didn't run. Check `ensureMigrations()` is awaited in `app/_layout.tsx` BEFORE rendering.
- **"unique constraint failed"** → trying to insert with an existing id. Use `.onConflictDoUpdate({ target: items.id, set: {...} })`.
- **`useLiveQuery` returns empty even after insert** → `openDatabaseSync` was called without `{ enableChangeListener: true }`. Fix in `lib/db/client.ts`.
- **Type error on `isDone`** — passing `0`/`1` instead of `false`/`true`. The `mode: "boolean"` field expects JS booleans.

## Performance

- SQLite handles thousands of rows fine. 100K+ → add indexes.
- Add an index: `index("items_user_done").on(items.isDone)` in the schema, then `db:generate`.
- Avoid `select *` on tables with text columns > 10KB. Project only what you need.
- `useLiveQuery` recomputes the WHOLE query on any change. For huge lists, paginate with `.limit().offset()`.

## Bridging to Supabase (sync engine)

The default `lib/db/sync.ts`:
1. Selects `where dirty = true` from local.
2. Upserts to Supabase via PostgREST.
3. Updates local: `serverId = returned, dirty = false`.
4. Selects from Supabase `where updated_at > localMax`.
5. Upserts into local with `dirty = false`.

For more than the default `items` table, extend `sync.ts` per-table OR generalize with a registry pattern. Don't add a sync layer — Drizzle gives you the primitives.
