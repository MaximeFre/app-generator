# Domain heuristics

One-pager per app vertical. Read by:
- `app-strategist` (during brief intake — informs default free/premium split, north-star metric, persona language)
- `app-architect` (during VISION — informs default IA shape, cadence, chart kinds, customization knobs, paywall split)

Different from `.claude/skills/design-research/categories/{vertical}.md` which is a research grid for the `design-researcher` agent (WebSearch questions). Domain heuristics here are **opinionated defaults** — what the architect should propose if the brief doesn't override.

## Files

- `fitness.md` — workout / training
- `journal.md` — mood / gratitude / habits
- `finance.md` — budget / expense / investing
- `productivity.md` — tasks / notes / time
- (add as needed: `health.md`, `social.md`, `dating.md`, `creative.md`, `utility.md`, `travel.md`)

## Format

Each file is ~80-150 lines:
- Default IA (tabs proposal)
- Default cadence
- Default chart kinds for stats screens
- Default customization knobs at point of use
- Default free vs premium split
- Default empty state vibe
- Vertical-specific anti-patterns
- North-star metric language

## Editing rules

These are defaults, not constraints. The architect can override when the brief calls for it. But if the brief is silent on something, these defaults apply.
